# 프로젝트 포트폴리오 정리 프롬프트

다른 프로젝트에서 아래 프롬프트를 복사해서 사용하세요. `[프로젝트 경로]` 부분만 수정하면 됩니다.

---

## 기본 프롬프트

```
[프로젝트 경로] 프로젝트를 포트폴리오·이력서용으로 정리해줘.

아래 형식에 맞춰서 문서(PROJECT_PORTFOLIO.md)를 작성해줘.

---

## 1. 프로젝트 개요
- 프로젝트명, 기간, 설명

## 2. 프로젝트에서 기여한 부분
- 내가 한 작업을 bullet point로 구체적으로 정리

## 3. 프로젝트를 해야 한 이유, 혹은 해결한 문제 정의

각 문제마다:
- **있어보이는 소제목** (예: 매칭 정보 분산 및 모집 플로우 단절, 실시간 경험 요구, 조회수 중복 카운팅, 사용자 액션-UI 반응성 간극)
- **간략한 내용** (1~2문장)
- **태그** (백틱으로 감싼 키워드들, 예: `원스톱 플랫폼` `스터디 1:N` `필터링`)

**구성 가이드:**
- 비즈니스적 문제: 플로우 차트 1개로 통합 (예: 스터디 생성→게시글 생성→모집)
- 기술적 문제: 3~4개로 추려서 각각 소제목·내용·태그 형식

## 4. 프로젝트의 문제 해결 과정 및 선택한 해결법 선택 이유

각 문제마다 **1. 문제인식 / 2. 해결 / 3. 결과** 3단계로 구성:

**1. 문제인식** — 있어보이는 소제목
간략한 내용.
`태그`

**2. 해결** — 있어보이는 소제목
간략한 내용 (어떤 기술/방법을 선택했는지).
`태그`

**3. 결과** — 있어보이는 소제목
간략한 내용.
`태그`

---

## 5. 결과(성과)

각 성과를 **카드 형식**으로:

### [성과 제목]
[한 줄 소개 문장]
- [주요 포인트 1]
- [주요 포인트 2]
- [주요 포인트 3]
`태그1` `태그2`

(카드 4~6개: 3-4번의 각 문제에 대응하는 결과 + 프로젝트 고유 성과 ex. AI 추천, 이미지 관리 등)

## 6. 문제인식 → 해결 → 결과 요약

## 참고: 겪었을 만한 이슈

- 하이드레이션, 인증, 성능 등 기술적 이슈 해결 과정

---

**지시사항:**
1. 실제 코드베이스를 분석해서 기술 스택·구현 내용 반영
2. 기억이 안 나는 부분은 "겪었을 만한" 이슈로 추론해서 적어줘
3. 3번(해결한 문제)·4번(해결 과정)·5번(결과)은 1:1 매핑 (같은 문제→해결→결과 순서)
4. SSR/SSG가 아닌 SSR/CSR로 표기
5. 확실하지 않은 라이브러리 선택이 있으면 물어봐
```

## README 업데이트 포함 프롬프트

```
[프로젝트 경로] 프로젝트를 정리해줘.

1. README.md 업데이트
   - 프로젝트 개요, 기술 스택, 디렉터리 구조, 시작 방법, 스크립트 정리

2. PROJECT_PORTFOLIO.md 생성 (위 형식 적용)
   - 3번: 있어보이는 소제목 + 간략한 내용 + 태그
   - 4번: 각 문제마다 1.문제인식 / 2.해결 / 3.결과 (소제목 + 내용 + 태그)
   - 5번: 카드 형식 (제목, 소개, 불릿 3개, 태그)

3. 코드베이스 분석해서 실제 구현 반영
4. 기억이 안 나는 기술 이슈는 추론해서 적어줘
5. 확실하지 않은 라이브러리 선택이 있으면 물어봐
```

## 짧은 버전 (빠른 요청용)

```
[프로젝트 경로] 포트폴리오용으로 정리해줘.

형식:
- 3번: 소제목 + 내용 + 태그 (비즈니스 1개, 기술 3~4개)
- 4번: 각 문제마다 1.문제인식 / 2.해결 / 3.결과 (소제목·내용·태그)
- 5번: 카드 형식 (제목, 소개, 불릿 3개, 태그)
- 3→4→5 번호 1:1 매핑

코드베이스 분석해서 실제 구현 반영. 기억 안 나는 이슈는 추론해서 적어줘.
```
